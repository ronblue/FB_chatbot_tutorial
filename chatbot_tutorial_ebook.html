there are many and different types of chatbots out there from the most basic simple ones to the most advanced and complex ones.

i would like to focus on several of them:

1. hard coded chatbots - chatbots that are completely fixed and hard coded from the top to bottom meaning that their keywords and replies are coded into the main code file and there for cannot be changed

2. databased chatbots - chatbots that their keywords and replies are stored on a database file (we will use in this tutorial a simple text file as such a database) these chatbots are more flexible to maintain or to change their keywords and replies since all we have to do is to change the database by editing them instead of changing the main source code of the chatbot program

3. learning chatbots - these chatbots have the ability to learn new keywords and replies and they do so by having usually databases (again a text file or actually two text files one for keywords and another one for replies) and when they are giving new input they don't know or remember they ask the user to teach them what replay should they replay to this new input and then store and add the new keywords (the new input) and the proper replay (new output) into their databases then next time they are given the same learned input they replay the learned replay (teaching such a bot can be at first a very boring or frustrating thing since at first they need to ask you alot what to replay it's like teaching a baby chatbot to talk from scratch)

4.(chatbots that will not be discussed in this tutorial) AI chatbots - these chatbot's are the most complex and advanced chatbot programs there is. they use machine learning and neural networks algorithms and usually have big data databases of conversations. they usually coded in c/c++ or python or java so they will not be discussed here.

that is the basic types of chatbots that i mention here of course there are other categories to chatbot's types like online bots like chatroom bots we see in chatrooms on discord or irc or mobile chatbots for android or iOS mobile phone apps and list go on and on...

hello friends here we will talk about the logic behind chatbots and what are they made of...

the most basic logic of the most basic chatbots is as the following:

well first of all the chatbot programs waits for input from the user. then it searches in that input for specific pre-defined KEYWORDS and if it finds any it will choose randomly from a specific pre-defined REPLIES a proper replay and will answer (meaning print) that replay to the user then it will wait for more input from the user and so on until the user gives a command to exit the program and then the programs end.

in addition to the KEYWORDS-REPLIES search there is also another set of DEFULT REPLIES that the chatbot program prints and answer to the user this happens when there are no keywords found in the input and they are generated randomly from a specific pre-defined DEFUALT REPLIES just in order for the chatbot to say something in any given time even if it didn't find any keywords in the user's input...

so far for the basic logic of how a chatbot program works but what are they made of?

well here are some of the objects or components chatbots are made of:

1. variables - especially string variables to store input from user and other matters

2. arrays - again arrays of strings they can be static arrays or dynamic arrays and they store the keywords and replies you want the chatbot to be able to identify and generate randomly

3. randomization - this is important for the replies of the chatbot to be as much originally so it will not repeat itself over and over

4. IF statements and SELECT CASE statements - for making the chatbot go over all the arrays of keywords and if the input contains them generate the proper random replay

5. for databased chatbots that uses text files as database for keywords and replies we also need to know how to OPEN the text file database for INPUT and read and load from it the keywords and replies and put then in the correct arrays

6. SUBS and FUNCTIONS - these are blocks of code that can be called and and executed whenever we need to use them instead of writing them over and over again and again these block of codes can get PARAMETERS to them which makes them flexible for re-use and they make a programmer's life much more easier

'load database file into arrays sub
SUB loadArrays(f AS STRING)
   DIM AS LONG h = FREEFILE()
   DIM AS STRING fileline
   OPEN f FOR INPUT AS #h
   WHILE NOT EOF(h)
      LINE INPUT #h, fileline
      SELECT CASE LEFT(fileline, 3)

            SAPPEND keywords1(), TRIM(MID(fileline, 4))
         CASE "r1:"
            SAPPEND replies1(), TRIM(MID(fileline, 4))
         CASE "k2:"
            SAPPEND keywords2(), TRIM(MID(fileline, 4))
         CASE "r2:"
            SAPPEND replies2(), TRIM(MID(fileline, 4))
         CASE "d1:"
            SAPPEND nokeyfound(), TRIM(MID(fileline, 4))
         CASE "e1:"
            EXIT while
      END SELECT
   WEND
   CLOSE #h
END SUB

FUNCTION check_input(txt AS STRING, txt_array() AS STRING) AS boolean
   FOR i AS INTEGER = 0 TO UBOUND(txt_array)

         RETURN TRUE
      ENDIF
   NEXT
   RETURN FALSE

   DIM h AS LONG = FREEFILE()

         LINE INPUT #h, fline
         SAPPEND ans(), fline
      WEND
   CLOSE #h
   OPEN questions FOR INPUT AS #h
      WHILE NOT EOF(h)
         LINE INPUT #h, fline
         SAPPEND ques(), fline
      WEND
   CLOSE #h
END SUB

'sub for printing properly the chatbot replies
SUB bot_speak(t AS STRING)
   PRINT "bot: " & t
   PRINT

'this sub is important it checks is user input is new or not

'if the input is new then it will ask the user to give the proper replay
'and it will store it in the databases and in the arrays
SUB answersQuestions(txt AS STRING)
var result = 0
dim as boolean Found = false
DIM h AS LONG = FREEFILE()
DIM inpt AS STRING
FOR i AS INTEGER = 0 TO UBOUND(ans)
	IF instr(txt, ans(i)) <> 0 OR txt = ans(i) THEN
		found = true
		BOT_SPEAK ques(i)
		EXIT FOR
	END IF
	NEXT i

		INPUT "enter new input: ", inpt
		PRINT : BOT_SPEAK inpt
		OPEN answers FOR APPEND AS #h
		PRINT #h, txt

SUB main()
